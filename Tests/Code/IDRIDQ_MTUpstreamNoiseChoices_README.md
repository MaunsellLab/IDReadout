# Replicated upstream-noise choices

Install all files in this package together into `Tests/Code`, replacing the
older copies of `makeIDRIDQMTTrialInputs.m` and
`factorUpstreamDirectionalNoise.m`. The upstream-noise calibration package and
the MT Gaussian forward core remain prerequisites.

This stage applies the calibrated upstream directional-noise model to the
actual IDQ and IDR rectangular-step noise trials.

- The same fixed dense candidate bank is used for all trials.
- Physical and internal directional inputs pass through the same MT tuning
  transformation.
- Candidate covariance sampling is only an algebraic shortcut for drawing the
  full upstream vector; no separate readout noise is introduced.
- Candidate outputs are rectified and a hard maximum is taken within each
  patch. The patch with the larger maximum is selected.
- Twenty independent choice replicates are generated by default.
- Only logical choices, trial probabilities, and aggregated winner frequencies
  are saved. MT and candidate arrays are not retained.
- Existing outputs are never overwritten.

The MAT file retains `sessionIndex`, `trialIndex`, the observed correctness,
and a logical `nTrials`-by-`nReplicates` synthetic correctness array for every
condition. This is sufficient for the later matched gain fits without saving
the much larger candidate activation arrays.

The summary outputs include:

- experimental and replicated-synthetic performance by dataset/offset;
- performance for each replicate;
- experimental and mean synthetic performance by session/offset;
- induced preferred-MT and preferred-readout noise SD, both in raw response
  units and in equivalent response-to-1%-preferred-coherence units;
- a three-panel PDF diagnostic.

Run tests:

```matlab
clear simulateIDRIDQUpstreamNoiseChoices ...
  simulateUpstreamChoiceCondition ...
  makeIDRIDQMTTrialInputs ...
  factorUpstreamDirectionalNoise
rehash

results=runtests("testMTUpstreamChoiceSimulation.m");
assertSuccess(results)
```

Generate choices inside the already calibrated upstream-noise run:

```matlab
simulation=simulateIDRIDQUpstreamNoiseChoices( ...
  contextUpstream,calibration,'NReplicates',20);
```

The simulation uses the existing calibration run and therefore does not call
`initializeSyntheticRun` again. Runtime is driven mainly by trial count and
replicate count; reducing `NReplicates` is appropriate for a smoke test, but
the inferential run should use the planned 20 replicates.
